Animaties (GSAP)

Stack & best-practice patterns.

Plugins

De core-plugins die je moet registreren:

import { gsap } from "gsap";
import { Draggable } from "gsap/Draggable";
import { ScrambleTextPlugin } from "gsap/ScrambleTextPlugin";
import { ScrollTrigger } from "gsap/ScrollTrigger";
import { ScrollSmoother } from "gsap/ScrollSmoother";

gsap.registerPlugin(Draggable, ScrambleTextPlugin, ScrollTrigger, ScrollSmoother);

Belangrijke animaties

  • Logo lamp pulse + micro-parallax
  • Lockscreen → desktop (fade + tilt)
  • Notification slide (15px from top)
  • Keyboard diagonal wave (10ms stagger)
  • Specs bubbles (draw line + fade text)

Recorder integratie

Start recording → play timeline → timeline onComplete → stopRecording() → download .webm

const master = gsap.timeline({ paused: true, onComplete: stopRecording });
// ... build timelines ...
await startRecording();
master.play();

Development

Hoe de code en animaties georganiseerd zijn.

Folder layout

├── docs/
├── src/
│   ├── components/
│   ├── animations/
│   ├── assets/
│   └── styles/
├── .github/
└── README.md

Best practices

  • Gebruik ES modules
  • GSAP timelines per feature (voidStart, voidUnlock, voidSpecs)
  • Minimal DOM layout thrash — gebruik transforms & opacity
  • Keys zijn individuele DOM nodes (.key) voor stagger control

Component-architectuur

Componenten zijn modulair en kunnen in isolation getest worden.

  • LaptopShell — casing, bezel, lid
  • KeyboardMatrix — keys grid (.key)
  • ScreenLayer — content area for desktop & apps
  • NotificationBubble — avatar + app-badge + text
  • TouchUnlock — trackpad indicator
  • SpecsBubble — marker + auto-draw line + text
Tip: keep animations inside `/src/animations` and import them in page-level controllers.
